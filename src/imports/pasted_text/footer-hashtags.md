13. Сквозная система #хэштегов
Каждый контент (точка на карте. Статья, объявление, история – имеет заданные тэги)

Хэштег   		Значение		   	  Пример объектов
#жильё		поиск жилья			квартиры, аренда, апартаменты
#школы		школы для детей 		детские сады, школы, секции
#коворкинг 		рабочие места 		коворкинги, офисы на день
#семья 		для переезда с семьёй 	мероприятия, школы, развлечения
#первый_месяц 	этап переезда		советы новичкам, соц. службы
#уезжаю тег

Идея: один хэштег → фильтрует контент по всем разделам
Пользователь переходит в раздел через хэштег или выбирает его в фильтрах → платформа автоматически подбирает все объекты, статьи и точки на карте, которые имеют этот тег.

Привязка к другим разделам
В разделе “Истории релокации”, “Найти опору” или карточки объявлений:
Автоматически показываются объекты с теми же хэштегами, по которым пользователь пришёл (=этап релокации)
Например, пользователь кликнул на #первый_месяц → внизу истории релокации показываются советы, коворкинги и жильё, подходящие для новичка
Визуальная реализация:
Хэштеги можно выводить как кнопки с цветом, нажатие фильтрует все разделы сразу

Можно сделать breadcrumb: пользователь видит “Вы сейчас в теме #первый_месяц → показываются соответствующие разделы”

Все разделы реагируют:
Поле	Фильтрация по тегу
много	Показывает только точки с #первый_месяц
Истории релокации	Показывает историю, отмеченные этим тегом
Найти опору	Показывает наставников/сообщества с этим тегом
Карточки бесплатных	Показывает только объявление, соответствующий тегу

Техническая реализация
Хранение тегов:
В базе для каждой точки/карточки/статьи хранится массив тегов
Пример (JSON):
JSON
{
  "id": 123,
  "title": "Детская школа в Дананге",
  "tags": ["#школы", "#семья", "#первый_месяц"]
}
Фильтрация:
На фронтенде при клике на хэштег отправляется запрос к API с фильтром по тегу
Бэкенд возвращает только объекты с совпадающими тегами
Связь между разделами:
Фронтенд получает текущий хэштег, и каждая секция (“карточка объявлений”, “карта”, “истории”) использует его для фильтрации своего контента
Оптимизация карты:
Даже при фильтре по хэштегу, применяем кластеризацию и lazy load, чтобы карта не перегружалась
