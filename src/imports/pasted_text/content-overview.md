 	72. Афиша событий
Очень простая. Есть фильтры по темам
Структура:
Верх:
•	Фильтр по датам: Дата\место\тип события Сегодня / Выходные /Неделя / Месяц
•	Приоритетность (сначала те которые ближе к дате)

 - Деловые (ИТ, бизнес, языковые)
 - Спорт и экскурсии
 - Для детей

•	Фото
•	Название
•	Дата + время
•	Стоимость (бесплатно/цена)
•	Кнопка “Записаться” = переход на сообщение организатору
•	Внизу … из Relo пойдут на встречу
Внутри:
•	Подробности
•	Организатор: иван
•	Его активности из профиля
•	Написать организатору
1 баннер рекламный внизу (место для него, а пока на его месте мотивационная картинка такого же размера)
Уже идут … из рело

Карточка:
•	название
•	дата
•	описание
•	контакт организатора или ссылка на форму записи

Внизу – «Тоже ищет компанию» - профили с хэш тегом ищу друзей
«Новые в городе» - профили новеньких

Логика как создаются и добавляются события
🔥 0. Главный принцип
👉 Пользователь ничего не постит вручную
👉 Система аккуратно фиксирует только значимые моменты
🧩 1. ОБЪЯВЛЕНИЕ
🔹 Триггер: Пользователь нажал → «Создать объявление»
🔹 Событие: Ищу квартиру в Дананге  📅 сегодня
🔹 Когда показываем: 👉 сразу после публикации
🔹 UX момент: После создания: Объявление опубликовано 🌿  
Это первый шаг твоего пути

🧩 2. НАЧАЛО ОБЩЕНИЯ
🔹 Триггер:Пользователь: написал кому-то ИЛИ получил ответ
🔹 Событие: Начал общение 📅 сегодня
🔹 Когда показываем: 👉 только 1 раз
❗ Важно: не показывать: с кем, сколько сообщений
💡 Анти-спам логика:  если сообщений > 1:   не создаём новые события
🧩 3. СДЕЛКА / РЕЗУЛЬТАТ (самый важный блок)
🔹 Триггер: Пользователь нажал: 👉 «Сделка завершена» ИЛИ 👉 «Я нашёл»
🔹 Событие: Нашёл жильё 🎉 Всё получилось 🤍
🔹 Когда показываем: 👉 после подтверждения
🔹 UX момент: Сразу после: Поздравляем с первой сделкой 🎉  Это начало твоего пути
❗ Нюанс: 👉 пользователь должен сам подтвердить (не автоматически!)
🧩 4. ОТЗЫВ
🔹 Триггер: Пользователь оставил отзыв
🔹 Событие: Оставил отзыв  «Всё прошло спокойно, спасибо»
🔹 Когда показываем: 👉 после отправки
❗ Нюанс: 👉 не публиковать без текста (иначе пусто и странно)
🧩 5. Остальные активности 
🔹 Триггер: Пользователь: написал Историю, ответил на вопрос, сделал метку на карте
🔹 Событие: выше

🔥 6. УМНАЯ ФИЛЬТРАЦИЯ (очень важно)
👉 Правило: максимум 1 событие / тип / день
Пример: ❌ Плохо:  Начал общение Начал общение Начал общение
✅ Хорошо: Начал общение
🧠 7. СВЯЗЬ С ЭТАПАМИ
👉 Логика: 1 событие → Этап 1   2–3 события → Этап 2   4–6 → Этап 3
💡 Важно:
Считаем только: объявления, сделки, отзывы, публикации
НЕ считаем: общение, сохранения
🔐 8. ПРИВАТНОСТЬ (логика по умолчанию)
👉 Автоматически: 
Объявления, публикации → публично  
Сделка → частично  
Общение → скрыто  
Отзыв → публично
❗ Плюс: Каждое событие → можно скрыть
⚙️ 9. УПРАВЛЕНИЕ СОБЫТИЯМИ
В профиле: ⋯ (три точки на карточке)
Открывается: Скрыть, Удалить, Сделать приватным
🌿 10. ИТОГ: как это работает вместе
Пользователь делает: Создал объявление → событие
Написал → 1 событие
Закрыл сделку → сильное событие
Оставил отзыв → доверие
