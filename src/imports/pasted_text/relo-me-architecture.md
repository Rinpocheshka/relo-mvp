1. Главная страница
Основная задача страницы — помочь релокантам быстро найти жильё, услуги, события и поддержку.
Общий принцип
•	Mobile-first
•	80% действий — в 2–3 тапа
•	Нет перегруженного home
•	Главное — ощущение системы, а не ленты

👉 Почему нет отдельной “Главной”?
Потому что главная = Объявления (это ядро).
Чем меньше абстракции — тем меньше путаницы.

Каждый этап регулирует свой набор скрытых хэштегов , которые учитывают все, что интересно на этом этапе. В интерфейсе: Хэштеги скрыты, но использовалась система для фильтрации контента во всех разделах платформы. Позже когда добавляются услуги – разделы – объявления -  у каждой  из них есть хэш тег и по нему система идентифицирует какие объвления показывать. И когда новый хеш тег появился – его нужно добавить в базу этапов. 

2️⃣ Как будет работать при входе на платформу и выборе этапа
1.	Пользователь выбирает этап:
o	Например, «Только приехал» - 
2.	Платформа объединяет этапы страницы и автоматически:
o	Собирает все объекты/точки/карточки/истории/наставников по скрытым хэштегам этих этапов
o	Выводит их по разделам:
	Найти опору → наставники/сообщества с тегами этапов
	Карточки объявлений → объявления с тегами этапов
	Люди – кому это тоже актуально
3.	Пользователь видит структурированную страницу, где каждый раздел подсвечен и отсортирован по логике этапов
Но! Если он переходит на «карточку» этапа уже на главной – меняется только последовательность шагов

________________________________________
4️⃣ Техническая поставка
1.	Хранение хэштегов:
o	Для каждого этапа в базе сохраняются теги массива:
{ 
  "stage": "Только приехал",   "tags": ["#жильё", "#школы", "#первый_месяц", "#советы"] }"Только приехал" "#жилё" , "#школы" , "#первый_месяц" , "#советы" ] }
2.	Генерация страниц этапов:
o	На фронтенде при клике на сцену → берём массив тегов этого этапа
o	Для каждого раздела делаем API-запрос с фильтром по этим тегам
o	Результаты строят готовые этапы страницы с контентом по разделам
3.	Выпадающий список интересов:
o	В пользовательском интерфейсе просто берём частичный набор тегов и показываем их как текстовые кнопки/чекбоксы.
o	Пользователь может нажать на интерес → фильтруются только соответствующие объекты внутри раздела
4.	Плюсы:
o	Пользователь получает готовый, структурированный контент без лишних поисков.
o	Можно легко обновлять хэштеги этапов без изменения пользовательского интерфейса.
o	Платформа остается динамичной и гибкой

2️⃣ Таймлайн (горизонтальный)
Горизонтальные карточки этапов (иконка + название)
Активный этап выделен цветом / не актвиные чб
главный контент ниже перестраивается под выбранный этап (только при первом заходе и потом при переходе пользователя на следующий)

3️⃣ Список шагов
На десктопе: под таймлайном вертикально
На мобильных: под таймлайном вертикально, свайп по шагам
4️⃣ Главный контент страницы (адаптивный)
Карточки с информацией, советы, ссылки, изображения
Динамически меняется в зависимости от выбранного этапа
Пример: при выборе “Только приехал” — показываются советы по аренде, адаптации, подключению интернета
⚡️ Адаптивность
Десктоп: горизонтальный таймлайн сверху, шаги под ним, контент ниже
Планшет: таймлайн горизонтальный, прокручиваемый, шаги вертикально
Мобильный: таймлайн вертикальный, шаги под активным этапом, свайп между этапами

1️⃣ Frame / Artboard
•	Размер: 360x640 px (мобильный экран)
•	Название: "Этапы релокации"
2️⃣ Этапы таймлайна
•	Горизонтальные блоки
•	Каждый этап:
o	Иконка этапа (например, дом → "Только приехал")
o	Название этапа (16-18px bold, кликабельно)
o	Под названием этапа список интересов согласно таблице (popup компонент)
3️⃣ Обновление главной страницы в соответствии с этапом
•	Заголовок: Соответствует таблице этапов
•	Разделы (каждый раздел как блок отображается в нужной последовательности, которая отражена в таблице «Этапы релокации», столбец 4 (приоритеты главной)):
1.	Люди – с теми же запросами что у пользователя
2.	Найти опору → наставники / сообщества с тегами этапа
3.	Карточки объявлений → карточки с тегами этапа
4.	Под названием таймлайна список вопросов к нему
4️⃣ Взаимодействие
•	Клик на этап таймлайна → он подсвечивается и меняется текст внизу (если первый вход после онбординга – меняется вся главная, если переключение уже на странице -только список шагов со ссылками 
•	Все разделы синхронизированы через хэштег этапа (работает только при первичном входе на главную)
5️⃣ Figma-ready подсказки
•	Этапы таймлайна → компоненты (копируются для всех этапов)
•	Разделы страницы этапа → фреймы с заголовком + контентом
•	Стрелки и связи → vector lines, легко редактируемые
