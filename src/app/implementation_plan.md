# Архитектура Новой Системы Личных Сообщений

Пользовательский интерфейс обмена сообщениями будет полностью переработан для создания более удобного UX (пользовательского опыта) без внезапных переходов на другие страницы.

## User Review Required
> [!IMPORTANT]
> Данный план предполагает создание единого глобального модального окна для быстрой отправки сообщений. Страница со списком всех диалогов (`/messages`) **уже существует** в проекте, но мы сделаем ее более заметной с помощью индикатора новых сообщений. 
> 
> Согласны ли вы с подходом добавления красного индикатора (бэйджа) счетчика непрочитанных прямо на нижнюю навигационную панель и верхнее меню (на иконку "Сообщения")?

## Proposed Changes

### Окно Быстрой Отправки Сообщения (Global Message Modal)
Будет создан глобальный контекст и компонент для модального окна написания первого (или быстрого) сообщения.
#### [NEW] `src/app/components/MessageComposeModal.tsx`
- Всплывающее окно (Modal) поверх интерфейса.
- Будет содержать: имя получателя, текстовое поле (текстареа) с лимитом в 1000 символов, и кнопку "Отправить".
- Интеграция с существующей RPC-функцией `get_or_create_chat` для гарантии наличия комнаты чата, после чего сообщение сразу сохраняется в базу, а окно закрывается с позитивным уведомлением-тостом "Сообщение отправлено".

#### [NEW] `src/app/MessageModalContext.tsx`
- Глобальный провайдер, оборачивающий все приложение. Это позволит из любого места приложения (`Profile.tsx`, `Афиша`, `Объявления`) вызывать функцию `openMessageModal(userId, userName)` без дублирования кода модального окна в каждом отдельном файле.

---

### Обновление существующих точек входа (Кнопки "Написать")
Кнопки на всех страницах перестанут пытаться перенаправить пользователя и начнут открывать созданное модальное окно.
#### [MODIFY] `src/app/components/Profile.tsx`
#### [MODIFY] `src/app/components/PeopleNearby.tsx`
#### [MODIFY] `src/app/components/EventDetailsModal.tsx`
#### [MODIFY] `src/app/components/AnnouncementDetailsModal.tsx`
- Замена `handleMessageClick` на вызов функции `openMessageModal` из нашего нового глобального контекста.
- Убираем все отладочные `alert()`.

---

### Индикатор Непрочитанных (Unread Badges)
Пользователь должен визуально понимать, что ему пришло сообщение. 
#### [MODIFY] `src/app/components/Layout.tsx`
- Добавление локального состояния `unreadCount`.
- Написание запроса к таблице `messages` для подсчета тех сообщений, где `is_read = false` и `sender_id != current_user_id` (на базе правил RLS, Supabase в целях безопасности посчитает только те сообщения, куда есть доступ).
- Подписка через `supabase.channel` на изменения в таблице `messages` для **моментального (в реальном времени)** обновления счетчика непрочитанных.
- Добавление красивого индикатора (красный кружок/счетчик) поверх иконки "Сообщения" на Десктопной и Мобильной навигационных панелях.

## Open Questions
- В существующей базе данных в таблице `chats` есть поле списка участников. Нужна ли возможность отправлять изображения в первом сообщении из модального окна, или ограничимся только текстом на 1000 символов? (В текущем плане предполагается **только текст**).

## Verification Plan

### Manual Verification
1. Зайти на `http://localhost:5174` двумя разными пользователями (например, в разных браузерах или инкогнито).
2. Зайти на страницу чужого профиля.
3. Нажать **Написать**. Проверить, что модальное окно красиво появляется, счетчик до 1000 символов работает, и нельзя отправить пустой текст.
4. Отправить текст. Проверить, что появилось успешное уведомление.
5. Во втором браузере моментально должен появиться красный бейдж над разделом "Сообщения".
6. Зайти в "Сообщения" вторым пользователем и прочитать текст, бейдж должен исчезнуть.
